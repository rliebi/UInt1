This is the Library TsenLee Project wiki.

Im Modul “User Interfaces 1” ist ein Miniprojekt zu absolvieren. Das Miniprojekt wird benotet und zählt zu 33% zur Modulschlussnote. Das Ziel des Projektes ist es, konkrete Erfahrungen mit Java Swing zu machen. Dafür gilt es ein User Interface für eine neue Bibliotheksoftware zu realisieren. Die Problem Domain und Business-Logik sind weitgehend vorgegeben. Wir erhielten als Vorgabe folgende Artefakte: Eine Persona, mehrere Szenarien und Wireframes und die Bewertungskriterien.

Besonderheiten

Autofokus:

Wenn Tastatureingaben getätigt werden, wechselt der Fokus automatisch ins entsprechende Suchfeld.

Tab-Funktion:

In der MasterView (Hauptfenster) Schaltet die Tabulator-Taste schnell zwischen den verschiedenen Reitern um.

Internationalisierung

Das Programm ist zur Zeit auf Englisch verfügbar. Die Internationalisierung und Lokalisierung sind aber Vorbereitet, die Begriffe müssten nur noch Übersetzt werden.

StandartButtons

Es wurden keine Standart Buttons definiert. Die Vollständige Bedienung per Tastatur kann in einer späteren Version umgesetzt werden.

Fehlermeldungen

Es wurde Bewusst auf Fehlermeldungen bzw. Popups verzichtet. Wenn eine Funktion nicht verfügbar ist, wird diese nicht ausgeführt oder der Knopf deaktiviert. Es wird angenommen, das Olga nicht daran erinnert werden muss, das ein Buch überfällig ist. Sie sieht das Orange Warnsymbol.

Performance

LibraryApp läuft Plattformunabhängig. Es wird Java 1.7 vorausgesetzt. Die Beste Performance haben wir unter Mac OS X beobachtet. Unter Windows kann die Applikation träge reagieren. Die Bibliothek setzt ab diesem Sommer Macs ein.

Vollständigkeit

Wir haben uns auf die Vorgegebenen Szenarien konzentriert. Die Applikation ist ohne Persistierung nicht einsatzfähig. WIR EMPFEHLEN NICHT DIESE APPLIKATION PRODUKTIV EINZUSETZEN.
